# Релизный чеклист (MVP Altcoinlog 2.0)

## Перед продакшеном

- [ ] Задать сильные секреты: `AUTH_SECRET`, при необходимости `INGESTION_CRON_SECRET`.
- [ ] Настроить `DATABASE_URL` на управляемый PostgreSQL.
- [ ] Выставить `NEXT_PUBLIC_SITE_URL` на канонический домен (HTTPS).
- [ ] Прогнать миграции: `npx prisma migrate deploy`.
- [ ] Создать реального редактора (не использовать дефолтный пароль сида).
- [ ] Проверить `robots.txt` и `sitemap.xml` на прод-домене.
- [ ] Проверить RSS: `/feed.xml`.
- [ ] Убедиться, что опубликованные URL открываются и JSON-LD валиден (Rich Results Test).

## После деплоя

- [ ] Smoke: логин в `/admin`, создание статьи, смена статусов.
- [ ] Smoke: ingestion на тестовом источнике, promote → редактирование → publish.
- [ ] Мониторинг ошибок (Sentry / Log drains) — по желанию.
- [ ] Бэкапы БД по расписанию.

## Производительность (базово)

- [ ] Проверить TTFB главной и страницы новости в продакшене.
- [ ] При росте трафика: кэширование ленты (ISR / `unstable_cache`) и CDN для статики.
